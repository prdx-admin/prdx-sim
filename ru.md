# Симулятор экосистемы Paradox

Агентная модель экономики протокола [PARADOX](https://github.com/prdx-admin/prdx-docs): токен PRDX и резерв TON, операции Mint, Burn и Mine. Этот репозиторий — та самая симуляция, на которую ссылается официальная документация.

## Официальная документация

Сначала имеет смысл прочитать спецификацию протокола (комиссии, ценообразование, механика Mine, связь с парадоксом Бернулли):

| **English** | [en/docs.md](https://github.com/prdx-admin/prdx-docs/blob/main/en/docs.md) · [raw](https://raw.githubusercontent.com/prdx-admin/prdx-docs/main/en/docs.md) |

Репозиторий документации: [github.com/prdx-admin/prdx-docs](https://github.com/prdx-admin/prdx-docs).

**Связь с этим проектом.** Формулы Mint и Burn в коде согласованы с документацией (комиссия в TON при Mint, в PRDX при Burn; обновление резерва и эмиссии). Mine реализует минимальную награду, геометрию наград, вероятности исходов, отрицательное матожидание и ограничение максимума относительно эмиссии. Значения по умолчанию совпадают с таблицей протокола: \(\phi_{\text{mint}}=\phi_{\text{burn}}=0.015625\), \(EV_{\text{mine}}=-0.125\), \(L_{\text{mine}}=0.125\).

**Обозначение цены.** В документации базовый курс часто записан как \(P_{\text{base}}=S/R\) (в примерах — PRDX на 1 TON). В симуляторе в графиках и расчётах используется **TON за 1 PRDX**: `резерв / эмиссия`, то есть \(R/S\) — то же состояние пула, обратные единицы.

**Что добавлено только в симуляторе.** Поверх правил протокола включены эвристические *агенты* (профили, эмоции, тренд) для исследования совокупного поведения; этого слоя нет в смарт-контрактах.

Краткое описание репозитория на английском: [README.md](README.md).

## Установка

Из корня репозитория:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e .
```

## Запуск ноутбука

```bash
jupyter notebook paradox_model.ipynb
```

Ячейка с импортом подставляет `src` в `sys.path`, если вы запускаете ноутбук из корня без `pip install -e .`.

## Что моделируется

- Операции **mint/burn** с комиссиями, как в протоколе  
- **Майнинг** с отрицательным математическим ожиданием и проверкой лимита относительно эмиссии  
- **Девять встроенных профилей** агентов (риск, терпение, склонность к майнингу, стратегии выхода и т.д.)  
- **Результаты**: вывод в консоль, файлы `paradox_system_results.png` и `paradox_roi_results.png`
